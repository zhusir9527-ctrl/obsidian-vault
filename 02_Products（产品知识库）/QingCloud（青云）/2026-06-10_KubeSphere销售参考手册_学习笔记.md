## 二、容器基础知识（话术素材）

| 概念 | 客户能听懂的解释 |
|------|----------------|
| 容器 vs 虚拟机 | 容器共享宿主机内核，无OS虚拟化损耗，启动快资源少 |
| DevOps | 开发+运维协作的文化，自动化软件交付全流程 |
| CI/CD | 代码提交→测试→发布的全自动流水线 |
| 微服务 | 大应用拆成小服务，各自独立开发部署，互不干扰 |
| K8s | 容器编排引擎，Google 20亿容器/周的经验产物 |
| Pod | K8s最小调度单元，可以理解为一个"最小作战单位" |

## 三、KubeSphere 产品核心卖点

1. **极简**：向导式UI，降低K8s学习成本
2. **完整**：应用管理+CI/CD+服务治理+监控日志+安全 全栈覆盖
3. **松耦合**：不绑架客户。可部署于物理机/虚拟机/云平台/混合环境
4. **全栈**：青云自研IaaS→存储→网络→容器全栈方案

## 四、关键功能

| 功能 | 卖点 |
|------|------|
| 多租户 | RBAC权限+LDAP/AD集成+单点登录 |
| 网络 | Calico/Flannel+hostnic（绑定云平台SDN）+物理交换机LB插件 |
| 存储 | Ceph/GlusterFS/NFS+青云CSI（块存储+NeonSAN） |
| 多集群 | 统一门户纳管不同版本/厂商K8s集群 |
| DevOps | CI/CD流水线+代码安全扫描+S2I/B2I |
| 微服务 | 基于Istio（非SpringCloud），无代码侵入，多语言支持 |

## 五、需求场景

| 场景 | 目标客户 | 方案优势 |
|------|---------|---------|

## 六、销售话术（核心）

### 商机把控四问

1. **"贵司目前哪个领域有容器化需求？"** → 判断客户对容器的了解程度
2. **"此次项目包括哪些功能？"** → 通过关键字（CI/CD/微服务/应用管理）匹配产品
3. **"预算多少？规模多大？"** → 评估商机大小
4. **"预算含硬件吗？"** → 纯软利润高，一体机金额大

### 常见问答11题

| 问题 | 回答要点 |
|------|---------|
| vs 友商优势？ | 全栈交付（IaaS→存储→网络），基于K8s（非Mesos/Swarm），全功能，Istio微服务 |
| 获取渠道？ | kubesphere.io开源 + kubesphere.qingcloud.com商业 |
| 推荐配置？ | 需客户提供负载规模才能推算 |
| 开源存储运维？ | 青云不负责，可选NeonSAN/QingCloud块存储 |
| 容器化改造？ | 青云不提供改造服务，可咨询支持，客户自己推动 |
| 微服务化？ | 同上，青云OpenPitrix团队可分享最佳实践 |
| 必须用云平台？ | 不，KubeSphere对底层无依赖 |
| 易捷升级高级版？ | 可以 |
| CI/CD上线？ | 高级版1.0.0（2018.12） |
| 为什么要上容器？ | 内部推动力+外部市场压力，提高资源利用率、快速迭代、标准化环境 |
| 容器安全？ | OS层隔离+镜像来源可控+安全扫描+RBAC |
