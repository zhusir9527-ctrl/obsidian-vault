## 一、前言

### 1.1 关于青云

青云智算平台由**北京青云智算科技有限公司**（青云科技控股子公司，青云科技股票代码：688316）打造。青云科技自2012年创立，坚持核心代码自研，2021年登陆上交所科创板（"混合云第一股"）。

青云科技定位为：
- 云原生技术核心服务商与引领者（集群管理、DevOps、可观测性、微服务）
- AI算力领域先行者与领导者
- 端到端数字化解决方案提供商

### 1.2 行业挑战

白皮书指出AI智算平台面临四大核心挑战：

1. **资源管理复杂，交付门槛高** — 异构芯片层（CPU/GPU/NPU）+ 高性能网络层（RoCE/IB）+ 海量数据层，CUDA版本错配、网络拓扑偏差、存储IO瓶颈均可导致训练中断
2. **异构算力调度整合难度大** — 大规模训练/推理场景下，如何管理异构资源、灵活调度
3. **快速形成应用场景困难** — 教学科研、算法开发、产业落地等多元场景的差异化算力需求
4. **算力运营效率低** — 硬件采购+电力+运维的高成本投入，需求波动导致资源闲置与紧张并存

## 二、智算平台介绍

### 2.1 平台简介

基于Kubernetes生态的一站式、可运营AI智算平台，核心价值：

| 层面 | 能力 |
|------|------|
| **资源层** | 异构算力统一纳管、算力切分与池化、精细化调度，提升资源利用率 |
| **用户层** | 多层级账号体系、精细化权限控制、资源配额管理 |
| **运营层** | 灵活定价策略、多维度账单、用量分析 |
| **生态层** | 支持主流框架、适配国产芯片、开放API接口 |

### 2.2 平台定位

- 自主可控、功能完备、智能运维、成熟运营
- 开源开放、安全合规、中立可靠、灵活部署
- 支持大规模模型开发、训练与推理
- 基于集群架构实现异构算力资源统一管理与弹性扩展

### 2.3 六大核心能力

1. **异构算力统一管理** — 将GPU/NPU/CPU统一映射为Kubernetes可调度对象，通过Device Plugin实现统一调度
2. **智能化调度算力** — 优先级、抢占、公平共享、Gang、Binpack/Spread、NUMA-aware等多种高级调度策略；支持任务排队、自动重试、故障自动迁移和断点续训
3. **故障监测与自愈** — 物理节点→计算卡→容器多层次监控；DCGM暴露显存利用率、SM占用率、PCIe重试次数、XID错误代码；支持企业微信/钉钉/Webhook多渠道告警；自动节点隔离
4. **一站式AI训推计算** — 贯通数据→开发→训练→推理→服务的AI全生命周期
5. **算力池化和算力切分** — 三层切分：MIG硬件级 + vGPU虚拟化层 + HAMI调度层（显存维度）
6. **全流程自服务式供给** — 账户开通→充值→资源申请→在线创建→资源释放，支持按需计费/包年包月

## 三、架构与关键技术

### 3.1 总体架构设计原则

**"资源统一抽象、调度集中决策、AI任务解耦执行"**

- 管理节点与计算节点分离
- 管理节点部署代理服务、前端服务、平台管理、可观测组件及K8s核心组件
- 计算节点通过自定义算力池区分不同场景
- 所有容器实例、分布式任务、模型部署推荐挂载高性能并行文件系统

### 3.2 技术架构分层

```
┌─────────────────────────────────────────────┐
│           AI 作业生命周期管理层              │
│   (PyTorchJob/TFJob/弹性容错/断点续训)        │
├─────────────────────────────────────────────┤
│         智能调度与资源管理层                  │
│   (拓扑感知/配额管理/优先级调度/Volcano)       │
├─────────────────────────────────────────────┤
│        异构算力接入与虚拟化层                 │
│  (Device Plugin/MIG/vGPU/HAMI/显存切分)      │
├─────────────────────────────────────────────┤
│       高性能数据与存储层                      │
│  (并行文件系统/Harbor镜像/P2P分发/Trivy扫描)  │
├─────────────────────────────────────────────┤
│          可观测性体系                        │
│  (DCGM/Prometheus/Grafana/OpenSearch)         │
└─────────────────────────────────────────────┘
```

### 3.3 关键技术详解

#### 3.3.1 云原生基础设施
- 以生产级Kubernetes集群为核心，结合KSE（KubeSphere Enterprise）提供企业级容器管理
- Istio Ingress Gateway统一流量入口
- Prometheus + Grafana + OpenSearch可观测体系

#### 3.3.2 算力池化与多租户隔离
- 公共资源池（多租户共享）+ 专属资源池（节点标签隔离）
- Kubernetes Namespace租户边界 + Network Policy网络隔离 + RBAC权限控制
- 容器运行时隔离

#### 3.3.3 异构算力统一调度
- 基于Volcano调度器：Gang/Priority/Fair-share/NUMA-aware
- 拓扑感知调度：结合NVIDIA Network Operator获取GPU物理拓扑
- 任务排队、自动重试、故障自动迁移、断点续训

#### 3.3.4 GPU切分技术（三层）
| 层级 | 技术 | 说明 |
|------|------|------|
| 硬件层 | NVIDIA MIG | 物理级隔离，单卡切分为多个独立实例 |
| 虚拟化层 | vGPU | GPU虚拟化与共享（需厂商驱动及License） |
| 调度层 | HAMI | 显存维度细粒度资源分配与共享 |

#### 3.3.5 网络架构
- Fat-Tree架构（1:1无收敛），多级CLOS网络
- 支持多轨接入（GPU服务器多网卡接入多台Leaf交换机）
- 支持InfiniBand或RoCE网络
- 管理网络/计算网络/存储网络三网分离

#### 3.3.6 部署架构
- 管理节点：3节点CPU物理机HA集群（etcd + API Server + HAProxy）
- 计算节点：支持不同型号GPU混部（A100/H800等）
- 节点内HAProxy本地负载分发 + 本地优先路由策略
- 支持按需横向扩展

## 四、产品功能（10大模块）

### 4.1 异构算力调度
- 统一纳管NVIDIA、昇腾、海光等国内外芯片
- 支持GPU池化、动态分配、弹性伸缩
- 兼容主流AI框架

### 4.2 vGPU虚拟化
- MIG物理切分 + vGPU虚拟化 + HAMI调度层切分
- 支持整卡/1/2/1/4/1/8等粒度
- 满足开发测试→推理→训练不同场景

### 4.3 存储服务
- 支持GPFS、Lustre、JuiceFS、ParaStor、OceanStor Pacific等第三方并行文件系统
- 每位用户默认20GB永久免费存储
- 存储独立于计算，容器销毁数据不丢失

### 4.4 容器实例
- 轻量级、弹性化、开箱即用的容器化计算环境
- 支持SSH/JupyterLab/VSCode多种访问方式
- 支持定时自动关机/释放
- 支持按需与包年包月互转、自动续费
- 支持pip/conda/apt安装源配置

### 4.5 分布式训练
- 集成PyTorch、TensorFlow、**MindSpore**、DeepSpeed、MPI等主流框架
- 任务模板化，一键创建
- 自动重试、超时控制
- 支持保存为模板、复制任务

### 4.6 推理服务
- 模型广场 + 模型管理 + 模型部署三大模块
- 支持文本/图像/视频/语音识别/嵌入/重排序等多模态模型
- 支持单节点与分布式两种部署方式
- 一键部署自动生成专属URL和API Key
- 支持手动在线扩缩容

### 4.7 大模型服务
- Web体验中心（多轮对话、系统提示词、推理参数调节、多模型对比）
- API密钥管理 + 用量统计 + Token计费
- 模型广场支持文生文/文生图/文生视频/图生视频/重排序
- 管理端支持模型排序、上下架、渠道配置

### 4.8 公共服务
- **镜像仓库**：Harbor企业级镜像管理，支持公共镜像+自定义镜像+P2P分发+Trivy安全扫描
- **专属资源组**：节点级管控、多维度调度策略、团队共享与隔离

### 4.9 用户管理
- 主账户/子账户多层级体系
- 用户组 + 角色 + 配额四维权限管控
- 支持子账户付费权限控制、余额共享

### 4.10 运营管理
- 账户管理、产品管理、规格管理、计费定价、费用中心
- 包年包月（预付费）+ 按需计费（后付费）
- 账单按多时间维度/多维度筛选/数据导出
- 账单数据每小时更新

## 六、平台关键特性总结

1. **云原生底座**：Kubernetes + KSE，容器化全栈
2. **异构算力统一纳管**：NVIDIA + 昇腾 + 海光等
3. **三层GPU切分**：MIG + vGPU + HAMI
4. **训推一体化**：开发→训练→微调→推理→部署全流程
5. **企业级多租户**：主账户/子账户/用户组/角色/配额
6. **灵活计费**：按需/包年包月，预付费/后付费
7. **可运营智算中心**：支持IDC/运营商商业化运营
8. **国产芯片适配**：华为昇腾（MindSpore + MindIE + vLLM-Ascend）、海光等
9. **高性能网络**：InfiniBand/RoCE，Fat-Tree拓扑
10. **智能运维**：DCGM深度监控、XID错误自愈、故障节点自动隔离
