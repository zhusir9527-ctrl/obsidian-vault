## 【算力前沿】

### DRAM涨价推高推理成本，大厂集体转向分层存储架构

DRAM价格已连续5个季度上涨。AI驱动的结构性短缺是根源。单台AI服务器内存配置是普通服务器的8到10倍，而全球约70%的先进存储产能被AI领域"吞噬"（TrendForce数据）。

但更值得关注的是涨价倒逼出的技术路线转向。

AMD正在研发AI调度技术，将推理中的"冷数据"自动迁移到闪存，仅在需要时调回DRAM。苹果则将模型常驻NAND闪存，通过自研芯片的快速读写能力弥补延迟差距。Marvell以硬件压缩技术扩大有效存储容量。闪迪推出HBF新型架构，试图在NAND上实现接近DRAM的性能。

这些方案的共同逻辑是：不再把所有数据塞进昂贵的内存，而是按"冷热分层"。活跃推理数据留在DRAM，低频数据放到SSD或归档存储，用软件调度弥补硬件差距。

对普通用户的影响：如果这些方案落地，AI推理的硬件成本将下降，最终体现为API定价的竞争空间。但短期来看，DRAM供给紧张至少持续到2027年（TrendForce预测），下半年API降价空间有限。

> 来源：华尔街见闻（2026/07/05）、TrendForce、Avnet市场分析（2026/02）

### 阿里巴巴内部禁用Claude Code，定性高风险软件

阿里内部将Anthropic的Claude Code定性为"高风险软件"，理由是存在"间谍软件隐患"和"后门风险"。这是继三星、SK等企业之后，又一家大规模禁止Claude Code的科技巨头。截至发稿，阿里巴巴未就此公开回应。

此前VentureBeat数据（7/3）显示，Claude Code在企业编程市场份额已达42%至54%，年化收入约25亿美元。阿里禁令可能引发更多中国企业效仿，影响Anthropic在亚洲的企业市场扩张。

对用户的影响：如果你在跨国企业或中资公司做开发，需要确认企业IT政策是否限制Claude Code使用。国内替代方案包括智谱ZCode（7/2发布）、通义灵码等。

> 来源：TechCrunch（2026/07/04）

## 【模型动态】

### GPT-5.5 Codex推理Token聚类异常，超60条社区反馈待修复

GitHub Issue #30364（7/4提交）报告，GPT-5.5 Codex在复杂推理任务中出现"推理Token聚类"异常。 `reasoning_output_token` 在516、1034、1552三个值处异常集中，导致长链推理任务在某几个步骤后性能断崖式下降。该问题主要影响长链推理（>2K tokens）场景，短任务不受影响。

该issue已获63条回复，OpenAI尚未确认修复时间线。社区猜测可能与推理Token预算的限制策略有关。

对用户的影响：如果你在用Codex处理多步骤推理任务（如代码生成、数据分析），建议主动分割长任务，避免触发Token聚类边界。

## 【今日Token概念解释】

### 存储分层架构（Tiered Memory Architecture）

简单说，就是把数据按"金、银、铜"分三级：贵的存高频数据，便宜的存低频数据。

**DRAM（金）** ：速度最快、价格最贵，存正在推理中的活跃数据。 **NVMe企业级SSD（银）** ：速度中等、价格适中，存推理冷数据（底层介质仍是NAND闪存）。 **QLC SSD或归档存储（铜）** ：速度最慢、最便宜，存模型权重和长期存档数据。

AMD的AI调度技术做的就是：自动识别哪些数据是"热的"（刚被推理引擎调用过），留在DRAM；哪些是"冷的"（很久没被用到），挪到SSD。这套逻辑在传统数据库和云计算中已经用了二十年，但应用到AI推理领域是近半年的新趋势。

对Token钱包的影响：如果分层存储普及，推理成本中的内存成本将下降。这意味着你的API调用费用有望在未来半年到一年内降低。
