# 统一文件命名规范

## 通用格式

```
YYYY-MM-DD_主题名称.md
```

## 示例

| 场景 | 文件名 |
|------|--------|
| 客户分析 | `2026-06-04_零跑汽车DNS需求分析.md` |
| 技术方案 | `2026-06-04_KubeSphere容器平台方案.md` |
| 竞品分析 | `2026-06-04_阿里云vs青云AI平台对比.md` |
| 会议纪要 | `2026-06-04_振石股份K8s项目推进会.md` |
| 投标复盘 | `2026-06-04_宁波水务虚拟化项目复盘.md` |
| 架构设计 | `2026-06-04_智算平台网络架构设计.md` |
| 学习笔记 | `2026-06-04_Kubernetes调度器源码分析.md` |
| 日报 | `2026-06-04.md` |

## 规则

1. 日期在前（便于排序）
2. 主题简洁明了
3. 中文命名（增强 RAG 语义理解）
4. 不使用空格，用下划线或直接连接
5. 版本号写主题中（v1/v2/final）

## 禁止

- ❌ 无日期前缀的文件名
- ❌ 含空格的文件名
- ❌ 纯英文缩写文件名（RAG不友好）
- ❌ `未命名.md` `新建文档.md`
