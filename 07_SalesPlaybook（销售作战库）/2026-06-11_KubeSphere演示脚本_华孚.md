## 脚本正文

---

### 第一阶段：第一印象（3分钟）

**页面：登录页 → 平台概览**

---

**话术：**

"徐总、各位老师好，我是青云的朱祚德。今天我给大家快速演示一下我们的AI智算平台。

（登录）

大家现在看到的是平台的首页。左边是导航栏、中间是资源概览、右边是快捷入口。设计理念就一个词——**开箱即用**。不需要背kubectl命令，所有操作可视化完成。

先看集群概览。这台服务器是8卡GPU，下面可以看到——

（指着仪表盘）

CPU使用率、内存、磁盘，特别看一下GPU——8张卡在线，显存使用情况一目了然。这个对华孚特别重要，因为你们多工厂的GPU分散管理，利用率不到30%，而我们的平台可以把所有GPU统一纳管、统一调度，利用率轻松提升到65%以上。"

**操作：**
1. 登录，进入平台首页
2. 点击「集群管理」→ 集群概览
3. 指出 GPU 监控区域
4. 鼠标悬停在 GPU 使用率上，显示详情

**切换语：** "集群层面看完了，接下来看我们怎么把资源分给不同团队用——这是华孚最需要的多租户能力。"

---

### 第二阶段：多租户（4分钟）

**页面：企业空间 → 项目管理**

---

**话术：**

"华孚有多个工厂、多个部门——配色团队、质检团队、供应链团队、IT运维。每个团队需要的资源和权限都不一样，混在一起会出问题。

我们的做法是建企业空间——

（点击企业空间）

比如我给大家演示：建一个「AI配色」的企业空间，给配色团队用。这里面我只授权一台GPU集群，设配额——比如8张GPU里分2张给配色，CPU 16核、内存32G。

（进入空间，点项目）

空间里面再建项目。比如配色测试环境、配色生产环境，每个项目还能再设配额。

关键是权限——配色团队的人只能看到「AI配色」这个空间，完全看不到质检团队或供应链团队的空间。配色团队的人在项目里是管理员，能部署模型、调参数；但如果给他只读权限，他只能看不能改。

这样华孚就能——一个平台，多个工厂、多个部门共用，互不干扰，资源不浪费。"

**操作：**
1. 点击「企业空间」，展示空间列表
2. 新建一个示例企业空间（或展示已有的）
3. 进入空间，查看配额设置
4. 进入项目列表，切换角色演示权限差异

**切换语：** "资源分好了，接下来看核心——怎么在这个平台上跑AI模型。"

---

### 第三阶段：模型部署（6分钟）★ 核心环节

**页面：模型广场 → 推理服务**

---

**话术：**

"这是我们的模型广场。

（点击模型广场）

相当于一个模型的应用商店。预置了Qwen、DeepSeek、LLaMA等主流开源模型。

部署一个模型有多简单？我演示一下——

（选择 Qwen2.5-7B，点击部署）

选模型 → 点部署 → 选要几张GPU → 确定。一分钟之后，这个模型就以API的形式对外提供服务了。

不需要改代码、不需要配CUDA、不需要手动下载模型——平台自动从ModelScope下载，自动配置推理引擎，自动生成API地址和密钥。

（等待部署完成，约30秒后）

好了，部署完成。现在这个模型已经有了一个API地址。

（打开API测试页面）

我调一下试试——用OpenAI兼容的格式，curl一行命令就能调用：

```
curl https://平台地址/api/v1/chat/completions \
  -H "Authorization: Bearer <api-key>" \
  -d '{"model":"qwen2.5-7b","messages":[{"role":"user","content":"请解释色纺纱的配色原理"}]}'
```

（展示返回结果）

OpenAI兼容意味着——华孚现有的应用、Agent、小助手，只要原来对接OpenAI的，改一行API地址就能直接用我们的私有模型，零迁移成本。"

**操作：**
1. 点击「模型广场」，浏览预置模型列表
2. 选择一个模型，点击部署
3. 选择GPU数量和节点，确认
4. 等待部署完成（<1分钟）
5. 展示 API 地址和密钥
6. 用 curl 或在线测试工具调一次 API，展示返回

**切换语：** "模型跑起来了。但真正用起来，华孚需要监控——Token用了多少、GPU负载怎么样、有没有异常。接下来看可观测性。"

---

### 第四阶段：可观测性（4分钟）

**页面：监控面板 → Token统计**

---

**话术：**

"平台提供两层监控。

第一层，GPU硬件监控——

（点击GPU监控）

每张卡的显存使用量、算力利用率、温度、功耗，实时刷新。可以设阈值——比如GPU温度超过80度，自动发企业微信告警。

第二层，模型调用监控——

（切到模型监控）

Token吞吐量——每秒处理多少Token。API请求次数、延迟、错误率。按小时、按天都能看。

（指Token用量面板）

这个对管理特别有用——比如配色团队这个月调用了100万Token，质检团队调用了50万。我可以看到每个团队、每个模型的用量趋势，资源分配有数据依据，不再是拍脑袋。

而且这些指标都可以导出——徐总向孙总汇报的时候，直接拉一张表，清清楚楚。"

**操作：**
1. 进入GPU监控页面，展示实时仪表盘
2. 演示告警策略配置
3. 切换到模型用量统计页面
4. 展示按团队/模型维度的Token统计
5. 导出报表（可选）

**切换语：** "最后看一下，如果华孚未来不只是用预训练模型，还想在自己的数据上做微调，怎么做。"

---

### 第五阶段：开发环境（3分钟）

**页面：容器实例 → Jupyter**

---

**话术：**

"这是开发环境。

（点击GPU计算模块 → 创建容器实例）

配色团队的算法工程师，需要一个GPU开发环境写代码、调模型——直接创建容器实例。

选2卡GPU、选Jupyter镜像、设定时7天自动释放——点确定。

（等待容器启动，约20秒）

进去之后就是一个完整的Jupyter Lab，预装了PyTorch、TensorFlow、CUDA。他可以在里面加载华孚自己的配色数据，调参、微调模型、保存镜像。

做完之后保存为镜像，推到镜像仓库，下次直接从这个镜像启动——环境完全一致，不用每次重新配。"

**操作：**
1. 创建容器实例（2卡GPU，Jupyter镜像）
2. 进入 Jupyter Lab
3. 演示 `import torch; torch.cuda.is_available()`
4. 展示镜像保存功能
5. 展示定时释放设置（防止资源浪费）

**切换语：** "以上是我们平台的核心能力。现在我总结一下，然后大家有什么问题咱们聊。"

## 可能被问到的问题 & 应答

| 问题 | 应答要点 |
|------|---------|
| "和阿里云PAI比有什么区别？" | "阿里PAI强在公有云场景。我们是私有化部署，数据不出厂，而且平台内置了多租户治理——配色/质检/供应链三个团队各自独立用同一个集群，PAI做不到。" |
| "安全性怎么样？支持等保吗？" | "数据全程不出企业网络。支持RBAC权限控制+操作审计日志+OPA安全策略，满足等保2.0要求。" |
| "扩容麻烦吗？以后加GPU怎么弄？" | "新GPU服务器上架→接入集群→自动识别→纳入统一调度。全程图形化操作，不中断现有业务。" |
| "支持国产GPU吗？比如昇腾？" | "支持。同一平台统一纳管NVIDIA和昇腾NPU，国产化适配已经完成（鲲鹏/飞腾/海光+麒麟/UOS/openEuler）。" |
| "PoC的配色模型什么时候能出结果？" | "交付团队已经在优化了。如果华孚能把历史配色数据给到我们，我们可以联合调优，预计X周内有明显提升。" |
