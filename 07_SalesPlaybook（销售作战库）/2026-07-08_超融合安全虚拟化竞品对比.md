# 超融合安全虚拟化竞品对比（2026-07-08）

> 口径定义：**安全虚拟化** = 超融合平台内建的软件定义安全组件（下一代防火墙 / IPS / WAF / 防病毒 / 微隔离 / 等保一体化）。该概念由 [[深信服]] HCI 定义并推广（aSV 计算 / aSAN 存储 / aNET 网络 / **安全虚拟化**）。

## 第一梯队 · 原生安全虚拟化（标配组件、最成熟）

| 厂商 | 产品 | 安全虚拟化实现 | 收费模式 | 威胁度 |
|------|------|--------------|---------|--------|
| [[深信服]] | 信服云 HCI | 安全虚拟化组件：AF 下一代防火墙虚拟化 / vWAF / IPS / 防病毒 / 微隔离 / 等保套件 | 组件独立 License，按功能模块(vFW/vWAF/防病毒各自计费)+按 CPU/节点，基础超融合价不含 | 高 |
| [[华为]] | FusionCube / FusionCompute | HiSecEngine 虚拟防火墙 + 微隔离 + 安全服务链，配 HiSec Insight | 安全套件按虚拟化实例或 CPU 授权，政企常打包 | 高 |
| [[新华三]] | H3C UIS | SecPath vFW 虚拟防火墙 + 微隔离 + U-Center 安全，等保合规套件 | 安全套件按实例/CPU 授权 | 高 |

## 第二梯队 · 有安全虚拟化能力（深度/生态或授权稍逊）

| 厂商 | 产品 | 安全虚拟化实现 | 注意点 |
|------|------|--------------|--------|
| [[VMware]] (Broadcom) | vSAN + NSX | NSX 分布式防火墙 + 微隔离（全球最强 SDS 安全） | 贵、Broadcom 授权混乱、不适信创 |
| [[Nutanix]] | AHV + Flow | Flow Network Security 微隔离 | 国内落地少、价高 |
| [[浪潮]] | InCloud Rail / 云海 | 安全组件 + 微隔离 + 浪潮云安全 | 政企信创可打 |
| [[云宏]] | WinCloud | 安全能力 + 等保对接 | 华南/医疗有基础 |
| [[ZStack]] | ZStack Cloud | 内建分布式防火墙 / 安全组 / 微隔离（逐步补齐） | 轻量，安全深度中等 |

## 第三梯队 · 安全靠外挂 / 生态对接（非内建安全虚拟化）

| 厂商 | 产品 | 实现 | 说明 |
|------|------|------|------|
| [[SmartX]] | SMTX OS | 对接第三方云安全 | 金融强，纯超融合定位，无自研安全组件 |
| [[云易捷]]（本方） | 云易捷 | 外挂 Agent(主机安全) + VLAN/安全组 | 老版本无内建安全虚拟化，等保靠新版本补 |

## 收费提示

安全虚拟化几乎都是**选配组件、独立 License**，不含在基础超融合报价里：
- 深信服：安全虚拟化按功能模块（vFW/vWAF/防病毒各自计费）+ 按 CPU/节点授权，单独一笔。基础价便宜，安全一加价格翻上去。
- 华为 / 新华三：安全套件按虚拟化实例或 CPU 授权，政企项目常打包进整体方案。

## 打单启示 / 应答口径

1. 客户硬性要"安全虚拟化内建"时，[[云易捷]] 正面打 [[深信服]] 吃亏——不在此赛道对轰。
2. 反打法：拆解深信服安全虚拟化的隐性成本（组件独立收费 / 绑定其安全生态 / 扩容再掏钱），突出青云"计算存储网络融合 + 信创适配 + 安全解耦"的开放性与 TCO 优势。
3. 等保驱动型客户：引导"青云超融合 + 第三方专业安全（奇安信/绿盟/深信服安全产品单买）"解耦方案，规避一体机锁定。
4. 价格话术：深信服基础价看着便宜，安全虚拟化按模块加价后总价翻上去；用整包 TCO 对比而非单看基础 License。
5. 第一梯队（深信服/华为/新华三）为内建安全虚拟化+等保一体，硬正面竞争；第三梯队（SmartX/青云）同属安全外挂路线，可错位竞争。
