# 青云AI智算整体解决方案 · 生态位分析与合作伙伴图谱

> 朱祚德 | 2026年6月 | 面向浙江制造业客户的整体方案生态构建

---

## 一、核心问题

**青云有什么？**
KubeSphere 容器调度平台 + AI智算平台（GPU调度/模型管理/Token运营）+ 云易捷一体机 + NeonSAN存储 + IaaS私有云/混合云

**青云缺什么？**
到客户那边推整体方案，客户要的不是"调度平台"，而是"AI基础设施全栈"。以下拆解6个关键生态位，每个都标注青云已有能力和缺失——以及可以拉谁一起做。

---

## 二、六大生态位全景图

```
                    ┌──────────────┐
                    │  行业应用层   │  ← ISV合作伙伴
                    ├──────────────┤
                    │  AI开发平台   │  ← 青云已有 + 生态补充
                    ├──────────────┤
                    │  模型服务层   │  ← 青云已有(Token运营)
                    ├──────────────┤
                    │  数据平台层   │  ← 需要合作伙伴
                    ├──────────────┤
                    │  算力调度层   │  ← 青云核心(KubeSphere)
                    ├──────────────┤
                    │  硬件基础设施 │  ← 硬件厂商合作伙伴
                    └──────────────┘
```

### 生态位一：硬件基础设施层

| 子项 | 青云现状 | 缺什么 | 推荐合作伙伴 |
|------|---------|--------|-------------|
| GPU服务器 | 无（不卖硬件） | 4090/A100/H100服务器供应 | **联想(联想凌拓)** / **浪潮** / **H3C新华三** / **超聚变** |
| 网络 | 无 | IB/RoCE高速互联 | **NVIDIA Mellanox** / **华为数通** |
| 存储 | NeonSAN | 并行文件存储( Lustre/GPFS) | **联想凌拓(NetApp)** / **杉岩数据** |
| 一体机 | 云易捷 | 更多GPU型号适配 | **浪潮(inMerge)** / **H3C(OneStor)** |

**推荐打法：**
- **首选合作伙伴：联想凌拓**——联想有服务器+NetApp有存储，且联想在浙江制造行业渠道强
- **次选：浪潮**——国产GPU服务器市占第一，有inMerge一体机线可与云易捷互补
- **硬件层面青云不做：** 拉硬件厂商进方案，青云做软件层，联合投标。青云不赚硬件差价，赚平台License+服务费

### 生态位二：算力调度层（青云核心，已达到）

| 能力 | 覆盖 | 说明 |
|------|------|------|
| 容器/K8s多集群管理 | ✅ KubeSphere | CNCF开源，中国贡献前三 |
| GPU统一调度 | ✅ AI智算平台 | MIG切分+时分复用+弹性调度 |
| 混合多云管理 | ✅ KubeSphere | 私有云+公有云+边缘统一纳管 |
| 多租户隔离 | ✅ | 配额管理、RBAC、网络隔离 |

**这个生态位不需要合作伙伴——青云自己就是主角。**

### 生态位三：数据平台层（最大缺口）

| 能力 | 青云现状 | 缺什么 | 推荐合作伙伴 |
|------|---------|--------|-------------|
| 数据标注平台 | ❌ 无 | 图像/文本标注工具 | **海天瑞声** / **Scale AI** / **Labelbox** / 自建简版 |
| 数据管理/版本 | ⚠️ 弱 | 数据集版本管理 | **DVC(开源)** / **LakeFS** / **格创东智** |
| 向量数据库 | ❌ 无 | Milvus/Qdrant/Weaviate | **Zilliz(Milvus)** / **Qdrant** |
| ETL/数据管道 | ⚠️ 弱 | 制造行业数据接入 | **数澜科技** / **Kettle(开源)** |

**推荐打法：**
- **向量数据库首选Zilliz(Milvus)**——国产开源，已有金融/制造客户案例，可与KubeSphere集成
- **数据标注：现阶段不强求**——制造企业AI项目初期数据量不大，可先用开源Label Studio + 华孚内部标注团队
- **建议：数据层整体轻量化**——不追求完整数据中台，用"向量库+开源工具"先跑通RAG和微调场景

### 生态位四：模型服务层（青云基本覆盖）

| 能力 | 青云现状 | 缺什么 | 推荐合作伙伴 |
|------|---------|--------|-------------|
| 模型推理(API) | ✅ vLLM/TGI集成 | — | 开源即可 |
| Token计费 | ✅ Token运营系统 | — | 青云自研 |
| 模型微调(SFT/LoRA) | ⚠️ 部署框架有，缺工具链 | 低代码微调界面 | **LLaMA-Factory(开源)** / 自研 |
| 模型评测 | ❌ 无 | 评测基准+AB测试框架 | **lm-evaluation-harness(开源)** / **Evidently AI** |

**推荐打法：**
- 推理+计费已有，不需要额外合作伙伴
- 微调工具链用开源LLaMA-Factory集成到平台，零成本
- 评测框架同理，开源集成

### 生态位五：AI开发平台层（需要补强）

| 能力 | 青云现状 | 缺什么 | 推荐合作伙伴 |
|------|---------|--------|-------------|
| Notebook开发环境 | ✅ Jupyter集成 | — | — |
| 分布式训练 | ✅ 支持PyTorch/TF | — | — |
| 实验管理(MLOps) | ❌ 无 | MLflow/Weights & Biases | **MLflow(开源集成)** / **W&B(商业)** |
| 工作流/Pipeline | ⚠️ 弱 | 可视化AI Pipeline编排 | **Kubeflow(开源)** / **ClearML** |
| 模型注册/版本 | ❌ 无 | 模型仓库 | **MLflow Model Registry** / **HuggingFace** |

**推荐打法：**
- MLOps用MLflow开源集成——社区最活跃，无需商业授权
- W&B适合大客户定制（收费），但不作为默认方案
- **关键：不需要独立MLOps厂商**——开源组件即可覆盖90%需求

### 生态位六：行业应用层（最大的差异化空间）

这是青云整体方案中最需要ISV合作伙伴的一层。青云提供AI基础设施，ISV提供行业know-how和应用。

| 行业 | 典型场景 | 青云能力 | 需要的ISV | 推荐合作伙伴 |
|------|---------|---------|----------|-------------|
| **纺织** | 配色/质检/排产 | 算力+模型部署 | 纺织行业AI应用 | **华孚自研** / **致景科技(百布)** / **凌迪Style3D** |
| **半导体** | EDA仿真/缺陷检测 | 算力+GPU调度 | 半导体行业ISV | **华大九天** / **概伦电子** / **芯华章** |
| **汽车零部件** | 视觉质检/预测维护 | 算力+边缘 | 工业视觉ISV | **阿丘科技** / **思谋科技** / **创新奇智** |
| **化工** | 安全生产/工艺优化 | 算力+私有化 | 化工行业ISV | **中控技术** / **和利时** / **石化盈科** |
| **通用制造** | 知识库RAG/客服 | 算力+模型 | 通用AI应用 | **钉钉AI** / **飞书AI** / **自研** |

**推荐打法：**
- **纺织行业：华孚就是最好的ISV**——华孚出行业数据+场景+品牌，青云出平台，联合打造纺织行业AI应用
- **其他行业：不追求独家绑定**——每个行业找1-2家ISV做联合方案，青云不做应用层，但提供"认证ISV + 青云平台"的打包方案
- **视觉质检：工业AI领域最成熟**——阿丘科技(清华系，融资到C轮)、思谋科技(港中文，融资到B轮)，都有成熟的工业视觉产品

## 四、到客户面前的"整体方案"怎么讲？

### 一张图讲清楚青云的定位：

```
客户看到的是：         青云提供的是：
┌──────────────┐      ┌──────────────────┐
│  行业应用     │ ←── │  认证ISV合作伙伴   │
├──────────────┤      ├──────────────────┤
│  AI开发平台   │ ←── │  KubeSphere+AI平台 │ ← 青云核心
├──────────────┤      ├──────────────────┤
│  模型服务     │ ←── │  Token运营系统     │ ← 青云核心
├──────────────┤      ├──────────────────┤
│  数据平台     │ ←── │  Milvus+LabelStudio│ ← 开源集成
├──────────────┤      ├──────────────────┤
│  硬件基础设施 │ ←── │  联想/浪潮/超聚变   │ ← 硬件合作
└──────────────┘      └──────────────────┘
```

### 核心话术：

> "青云不做硬件，不做行业应用——我们做好AI基础设施最核心的调度和管理平台。硬件我们拉联想一起供，行业应用我们拉行业ISV一起做。青云的角色是'AI算力操作系统'，让所有硬件和应用都跑在我们上面。"

### 与纯阿里云/华为云的差异化：

| | 阿里云 | 华为云 | **青云+生态** |
|--|--------|--------|-------------|
| 硬件 | 阿里自有/合作伙伴 | 华为自有为主 | **联想/浪潮/超聚变，客户自选** |
| 平台 | ACK(锁定) | CCE(锁定昇腾) | **KubeSphere(开源不锁定)** |
| 行业ISV | 阿里生态内ISV | 华为生态内ISV | **开放生态，客户可带自有ISV** |
| 部署 | 公有云为主 | 混合云 | **私有化/混合云/边缘全支持** |
| 核心差异 | 生态强但锁定深 | 全栈但绑定重 | **开放+不锁定+联合运营** |
