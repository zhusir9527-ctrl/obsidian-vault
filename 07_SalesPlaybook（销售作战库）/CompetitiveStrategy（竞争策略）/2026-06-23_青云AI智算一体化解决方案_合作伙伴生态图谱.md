## 一、青云有什么、缺什么

| 层 | 青云自有产品 | 能力 | 缺什么 |
|----|------------|------|--------|
| 容器平台 | **KubeSphere** | K8s多集群/多租户/DevOps | — |

| 数据 | ❌ 无 | — | 向量库/数据标注/数据治理 |
| 行业应用 | ❌ 不做应用 | — | 质检/预测/排产/知识库 |
| 安全 | ⚠️ 基础能力 | 等保三级/国密 | 数据安全/零信任 |

## 三、推荐合作伙伴清单

### Tier 1 — 必须拉入生态（联合投标、联合方案）

| 伙伴 | 领域 | 合作内容 | 联系方式/线索 |
|------|------|---------|-------------|
| **联想凌拓** | 服务器+存储 | 联想出GPU服务器(4090/A100/H100)，NetApp出存储，青云出KubeSphere+AI智算平台。三方联合投标，青云不赚硬件差价 | 联想浙江政企部 |
| **Zilliz(Milvus)** | 向量数据库 | RAG知识库方案的标配组件。开源免费，集成到青云AI平台上，客户零额外成本 | 已融资到B+轮，上海 |
| **阿丘科技** | 工业AI视觉 | 制造业缺陷检测/OCR/定位。清华系，融资到C轮。产品成熟，可直接联合投标 | 北京/苏州，有浙江案例 |

### Tier 2 — 按项目拉入（特定场景、特定客户）

| 伙伴 | 领域 | 合作场景 | 为什么不是Tier 1 |
|------|------|---------|-----------------|
| **思谋科技** | 工业AI+视频分析 | 汽车零部件/电子制造复杂质检 | 偏高端，定价高，中小企业客户不一定需要 |
| **中控技术** | 化工/流程工业 | 化工企业(巨化/新安/传化)的工艺优化+AI | 只覆盖化工行业，非通用 |
| **浪潮信息** | 服务器 | 客户指定国产服务器品牌时 | 联想凌拓覆盖了主要场景 |
| **H3C新华三** | 服务器+网络 | 政企客户指定H3C品牌时 | 同上 |
| **数澜科技** | 数据中台 | 大客户(华孚级别)的制造数据治理 | 轻量级项目不需要，太重 |

### Tier 3 — 开源集成（零成本，但要做好适配）

| 组件 | 用途 | 集成到 |
|------|------|--------|
| **vLLM** | 模型推理加速 | AI智算平台（已有） |
| **LLaMA-Factory** | 模型微调工具 | AI智算平台（建议集成） |
| **MLflow** | 实验管理+模型注册 | KubeSphere上部署 |
| **Label Studio** | 数据标注 | 客户自用，平台提供部署模板 |

## 四、面对不同客户的"组合拳"

### 场景A：有GPU、有场景、缺平台 — 如华孚

```
客户已有：GPU服务器 + 行业数据 + 客户关系
青云出：  KubeSphere + AI智算平台 + 联合运营
ISV出：   不需要（客户自己就是ISV）
硬件出：   不需要（客户已有GPU）
```

### 场景B：有场景、缺GPU、缺平台 — 如士兰微

```
客户已有：行业know-how + 业务场景 + 预算
联想出：  GPU服务器 + 存储
青云出：  KubeSphere + AI智算平台
ISV出：   华大九天（EDA仿真）/ 阿丘科技（晶圆检测）
```

### 场景C：有预算、什么都没 — 如中小制造企业

```
青云出：  云易捷一体机（预装KubeSphere+AI智算平台）
ISV出：   阿丘科技（质检）+ LLM（通用大模型）
硬件出：  一体机内含（无需额外采购）
```

### 场景D：政府/园区AI算力中心

```
联想/浪潮出：批量GPU服务器
青云出：    KubeSphere + AI智算平台 + Token运营（多租户）
ISV出：     各行业ISV入驻平台
```

## 六、客户面前的整体方案话术

> **"青云的定位是AI算力操作系统——我们不做硬件，不做行业应用。硬件我们拉联想一起供，质检我们拉阿丘科技一起做，向量库用开源的Milvus。青云做好中间最核心的调度和管理平台。客户得到的是一套完整的、可落地的方案，不是一堆需要自己拼凑的零件。"**
