# 售前全流程工作流

> 从客户名称输入到方案话术输出的端到端自动化工作流。覆盖情报采集、分析洞察、方案设计、话术生成四大阶段，实现售前准备的系统化与智能化。

---

## 工作流全景

```
客户名称输入
    │
    ▼
┌──────────────────────────────────────────────────────────────────┐
│ 阶段一：情报采集                                                  │
│ AccountResearch ──→ CustomerIntelligence                         │
│ (客户研究)          (动态情报)                                     │
│ 输出：客户档案      输出：情报报告                                   │
└──────────────────────────────┬───────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│ 阶段二：分析洞察                                                  │
│ Discovery ──→ 竞品分析                                            │
│ (需求探询)     (竞争态势)                                          │
│ 输出：需求框架  输出：竞品报告                                       │
└──────────────────────────────┬───────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│ 阶段三：方案设计                                                  │
│ SolutionArchitect ──→ KSEArchitect ──→ AIPlatformArchitect       │
│ (主方案)            (行业方案)         (AI平台方案)                   │
│ 输出：技术方案书    输出：KSE方案包     输出：AI平台架构                │
└──────────────────────────────┬───────────────────────────────────┘
                               │
                               ▼
┌──────────────────────────────────────────────────────────────────┐
│ 阶段四：话术与策略                                                │
│ DealStrategy ──→ 拜访话术生成                                      │
│ (赢单策略)       (沟通话术)                                         │
│ 输出：策略文档    输出：话术包                                        │
└──────────────────────────────────────────────────────────────────┘
                               │
                               ▼
                          交付物汇总
              （客户画像 + 方案书 + 话术包 + 策略文档）
```

## 阶段一：情报采集

### 步骤1.1：客户研究（AccountResearch）

| 属性 | 描述 |
|------|------|
| 触发方式 | 输入客户名称后自动触发 |
| 执行Agent | `AccountResearch` |
| 输入 | 客户名称、行业类型 |
| 预计耗时 | 2-5分钟 |

**输入示例：**
```yaml
customer_name: "XX银行"
industry: "金融"
research_depth: "deep"
```

**输出：**
- 企业概况报告
- 财务与技术现状分析
- 组织架构推断
- 痛点假设清单

**输出文件路径：**
`03_Customers（客户知识库）/{{客户名称}}/客户档案_{{日期}}.md`

### 步骤1.2：动态情报采集（CustomerIntelligence）

| 属性 | 描述 |
|------|------|
| 触发方式 | 步骤1.1完成后自动触发 / 每周定时 |
| 执行Agent | `CustomerIntelligence` |
| 输入 | 客户名称、1.1输出 |
| 预计耗时 | 1-3分钟 |

**输出：**
- 购买信号识别
- 风险信号预警
- 竞品活动监测
- 舆情分析

**输出文件路径：**
`03_Customers（客户知识库）/{{客户名称}}/情报报告_{{日期}}.md`

### 阶段一完成标记
- [ ] 客户档案已生成并审核
- [ ] 情报报告已生成
- [ ] 异常信号已标注（如有）

---

## 阶段二：分析洞察

### 步骤2.1：需求探询（Discovery）

| 属性 | 描述 |
|------|------|
| 触发方式 | 阶段一完成后触发 / 拜访前48小时触发 |
| 执行Agent | `Discovery` |
| 输入 | 客户档案 + 情报报告 + 拜访对象角色 |
| 预计耗时 | 3-5分钟 |

**输入示例：**
```yaml
customer_name: "XX银行"
meeting_attendees: ["CIO-张总", "技术总监-李总"]
meeting_type: "initial"
solution_focus: "数据中台"
```

**输出：**
- SPIN探询框架
- 角色专项问题清单（5-8个/角色）
- 价值假设陈述
- 异议预判与应对
- 拜访议程建议

**输出文件路径：**
`07_SalesPlaybook（销售作战库）/{{客户名称}}/需求探询框架_{{日期}}.md`

### 步骤2.2：竞品分析

| 属性 | 描述 |
|------|------|
| 触发方式 | 与2.1并行 / 客户提及竞品时触发 |
| 执行Agent | 调用[[2026-06-04_售前场景Prompt库#2-竞品分析Prompt|竞品分析Prompt]] |
| 输入 | 竞品名称、项目需求、我方优势 |
| 预计耗时 | 2-4分钟 |

**输出：**
- 功能对比矩阵
- 优劣势分析
- 竞标策略建议

**输出文件路径：**
`04_Bids（投标项目库）/{{项目名称}}/竞品分析_{{日期}}.md`

### 阶段二完成标记
- [ ] 需求探询框架已生成
- [ ] 竞品分析已完成（如适用）
- [ ] 价值假设已与销售团队对齐

---

## 阶段三：方案设计

### 步骤3.1：技术方案生成（SolutionArchitect）

| 属性 | 描述 |
|------|------|
| 触发方式 | 阶段二完成后触发 / 招标文件收到后触发 |
| 执行Agent | `SolutionArchitect` |
| 输入 | 需求框架 + 产品知识库 + 技术约束 |
| 预计耗时 | 5-10分钟 |

**输出：**
- 架构概览
- 模块设计方案
- 集成方案
- 实施路线图
- BOM清单

**输出文件路径：**
`05_Solutions（解决方案库）/{{客户名称}}/技术方案书_{{日期}}.md`

### 步骤3.2：行业方案定制（KSEArchitect）

| 属性 | 描述 |
|------|------|
| 触发方式 | 3.1完成后按需触发 |
| 执行Agent | `KSEArchitect` |
| 输入 | 技术方案 + 行业需求 |
| 预计耗时 | 3-5分钟 |

**输出：**
- 行业化架构适配
- KSE组件规格
- 部署拓扑
- 最佳实践

**输出文件路径：**
`05_Solutions（解决方案库）/{{客户名称}}/KSE方案_{{KSE类型}}_{{日期}}.md`

### 步骤3.3：AI平台方案（AIPlatformArchitect）

| 属性 | 描述 |
|------|------|
| 触发方式 | 客户需求涉及AI/ML时触发（与3.2并行） |
| 执行Agent | `AIPlatformArchitect` |
| 输入 | KSE方案 + AI需求 |
| 预计耗时 | 3-5分钟 |

**输出：**
- AI平台架构
- 模型服务拓扑
- MLOps流程
- GPU集群设计
- 成本估算

**输出文件路径：**
`06_Architecture（架构设计库）/{{客户名称}}/AI平台架构_{{日期}}.md`

### 阶段三完成标记
- [ ] 技术方案书已完成
- [ ] 行业方案已定制（如适用）
- [ ] AI平台方案已设计（如适用）
- [ ] 方案已通过技术评审

---

## 阶段四：话术与策略

### 步骤4.1：赢单策略（DealStrategy）

| 属性 | 描述 |
|------|------|
| 触发方式 | 阶段三完成后触发 |
| 执行Agent | `DealStrategy` |
| 输入 | 技术方案 + 竞品分析 + 关系图谱 |
| 预计耗时 | 3-5分钟 |

**输出：**
- 赢单概率评估
- Battle Card
- 定价策略
- 行动计划
- 关键成功因素

**输出文件路径：**
`07_SalesPlaybook（销售作战库）/{{客户名称}}/赢单策略_{{日期}}.md`

### 步骤4.2：拜访话术生成

| 属性 | 描述 |
|------|------|
| 触发方式 | 拜访前24小时触发 / 随4.1并行触发 |
| 执行Agent | 调用[[2026-06-04_售前场景Prompt库#5-拜访话术生成Prompt|拜访话术生成Prompt]] |
| 输入 | 客户画像 + 需求框架 + 拜访角色 |
| 预计耗时 | 2-4分钟 |

**输出：**
- 开场话术
- 探询话术
- 方案呈现话术
- 异议处理话术库
- 收尾与跟进话术

**输出文件路径：**
`07_SalesPlaybook（销售作战库）/{{客户名称}}/拜访话术_{{角色}}_{{日期}}.md`

### 阶段四完成标记
- [ ] 赢单策略已生成
- [ ] 拜访话术已准备
- [ ] 话术已演练
- [ ] 交付物已汇总

## 自动化触发条件总览

| 触发条件 | 执行动作 | 频率 |
|----------|----------|------|
| 新建客户档案 | AccountResearch | 即时 |
| 客户信息>90天未更新 | AccountResearch 刷新 | 条件触发 |
| 每周一 09:00 | CustomerIntelligence 全量扫描 | 每周 |
| 客户进入销售漏斗 | CustomerIntelligence 深度采集 | 即时 |
| 拜访前48小时 | Discovery | 条件触发 |
| 客户提及竞品 | 竞品分析 | 即时 |
| 需求确认 | SolutionArchitect | 即时 |
| 招标文件收到 | SolutionArchitect + 标书应答 | 即时 |
| 涉及行业定制 | KSEArchitect | 条件触发 |
| 涉及AI/ML需求 | AIPlatformArchitect | 条件触发 |
| 进入竞争阶段 | DealStrategy | 即时 |
| 拜访前24小时 | 拜访话术生成 | 条件触发 |

```markdown

日期：{{date}}

## 客户画像
- [[客户档案_{{date}}]] — AccountResearch输出

## 情报分析
- [[情报报告_{{date}}]] — CustomerIntelligence输出
- [[竞品分析_{{date}}]] — 竞品分析

## 需求与策略
- [[需求探询框架_{{date}}]] — Discovery输出

## 技术方案
- [[技术方案书_{{date}}]] — SolutionArchitect输出
- [[KSE方案_{{type}}_{{date}}]] — KSEArchitect输出

- [[拜访话术_{{role}}_{{date}}]] — 话术生成
```

## 使用说明

### 启动工作流
在Obsidian中输入：
```
@EROS 启动售前工作流 {{客户名称}}
```

### 查看进度
工作流引擎会实时更新每个阶段的完成状态。

### 中断与恢复
- 每个阶段独立运行，支持断点续跑
- 阶段输出持久化存储，可通过引用链接直接跳转

### 自定义扩展
- 可在任一步骤后插入自定义处理节点
- Agent的触发条件可在工作流配置中修改
