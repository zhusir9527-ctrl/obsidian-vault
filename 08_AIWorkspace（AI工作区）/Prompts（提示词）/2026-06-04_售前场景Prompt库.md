## 1. 客户画像生成Prompt

### 适用场景
在接触新客户或开展售前准备时，基于已知信息（行业、规模、业务痛点、技术栈等）生成结构化的客户画像。

### Prompt模板

```
你是一位资深的企业级售前咨询顾问，擅长快速构建客户画像和需求预判。请基于以下信息，生成一份结构化的客户画像报告。

## 输入信息
- 客户名称：{{customer_name}}
- 所属行业：{{industry}}
- 企业规模：{{company_size}}（员工数/营收规模）
- 已知痛点：{{known_pain_points}}
- 现有技术栈：{{current_tech_stack}}
- 项目背景：{{project_background}}
- 关键决策人：{{key_stakeholders}}

## 输出要求
请按照以下结构输出客户画像：

### 1. 企业概况
- 核心业务描述
- 行业地位分析
- 数字化成熟度评估（1-5级）

### 2. 业务痛点分析
- 表层痛点（客户已表达的需求）
- 深层痛点（潜在未被明确表达的需求）
- 痛点紧急程度评估（高/中/低）

### 3. 技术现状评估
- 现有技术架构梳理
- 技术债务识别
- 与行业标杆的技术差距分析

### 4. 决策链分析
- 组织架构推断
- 关键角色与影响力评估
- 预算决策路径分析

### 5. 机会与风险
- 核心机会点（我们最能发挥优势的领域）
- 潜在风险（竞品优势、预算限制、政治因素等）
- 差异化切入策略建议

### 6. 下一步行动建议
- 建议的首次接触策略
- 需要进一步验证的信息
- 推荐的技术方案方向

请确保分析深入、逻辑严密，避免空洞套话。
```

### 使用说明
- 将 `{{}}` 中的变量替换为实际信息
- 如某些信息未知，保留变量标记或标注"待调研"
- 输出可用于[[客户档案模板]]填充
- 建议搭配 [[2026-06-04_EROS_Agent配置#CustomerIntelligence|CustomerIntelligence Agent]] 使用

## 2. 竞品分析Prompt

### 适用场景
在项目投标前或客户提出竞品对比需求时，对主要竞争对手进行系统化分析。

### Prompt模板

```
你是一位企业级市场竞争分析专家，请基于以下信息，生成一份全面的竞品分析报告。

## 输入信息
- 竞品公司名称：{{competitor_name}}
- 竞品核心产品：{{competitor_product}}
- 目标客户行业：{{target_industry}}
- 当前项目需求：{{project_requirements}}
- 我方核心优势：{{our_strengths}}
- 已知竞品策略：{{known_competitor_strategy}}

## 输出要求
请按照以下结构输出竞品分析：

### 1. 竞品概况
- 公司背景与市场份额
- 产品定位与核心卖点
- 在目标行业的典型案例

### 2. 产品功能对比矩阵
| 维度 | 我方 | {{competitor_name}} | 差异分析 |
|------|------|---------------------|----------|
| 核心功能 | | | |
| 技术架构 | | | |
| 部署方式 | | | |
| 扩展性 | | | |
| 安全合规 | | | |
| 售后服务 | | | |
| 价格模式 | | | |

### 3. 我方优势分析
- 技术领先领域
- 服务差异化优势
- 行业经验壁垒

### 4. 竞品优势分析
- 竞品最强领域（诚实评估）
- 竞品可能的攻击策略
- 客户可能提出的质疑点

### 5. 竞标策略建议
- 正面交锋策略（正面PK的场景）
- 侧面包抄策略（差异化定位）
- 避实就虚策略（回避竞品强项的话术）
- 关键的"一票否决"话术

### 6. 风险评估与应对
- 丢标风险等级（高/中/低）
- 关键风险点及应对预案
- 需要高层支持的资源申请

请以"赢单"为导向，既要客观又要策略性。
```

### 使用说明
- 可同时对多个竞品运行此Prompt，然后横向对比
- 功能对比矩阵建议填入真实数据和客户案例
- 搭配 [[2026-06-04_EROS_Agent配置#DealStrategy|DealStrategy Agent]] 使用

## 3. 技术方案生成Prompt

### 适用场景
在客户需求明确后，生成面向客户的定制化技术方案文档。

### Prompt模板

```
你是一位资深解决方案架构师，请基于以下客户需求信息，生成一份专业的技术方案建议书。

## 输入信息
- 客户名称：{{customer_name}}
- 项目名称：{{project_name}}
- 核心需求：{{core_requirements}}
- 技术约束：{{technical_constraints}}
- 预算范围：{{budget_range}}
- 实施周期：{{timeline}}
- 已有系统：{{existing_systems}}
- 我方产品/方案：{{our_solution}}
- 客户技术偏好：{{tech_preference}}

## 输出要求
请按照以下结构输出技术方案：

### 1. 项目理解
- 对客户业务目标的理解
- 对技术需求的解读
- 方案设计理念

### 2. 总体架构设计
- 架构概览图（文字描述+关键组件）
- 技术选型说明
- 设计原则（高可用、可扩展、安全性等）

### 3. 功能模块设计
- 模块1：{{module_name}} - 功能说明、技术实现、价值点
- 模块2：...
- （根据需求生成3-6个核心模块）

### 4. 关键技术方案
- 数据架构设计
- 集成方案（与现有系统的对接）
- 安全方案（认证、授权、加密、审计）
- 高可用与灾备方案

### 5. 实施计划
- 分阶段实施路线图
- 关键里程碑
- 风险与应对措施

### 6. 产品/方案优势
- 为什么选择我们（3-5个核心优势）
- 与替代方案的对比
- 成功案例佐证

### 7. 服务与支持
- 实施服务内容
- 培训计划
- 运维支持承诺

请确保方案技术深度足够、逻辑清晰、可落地执行。
```

### 使用说明
- 可根据客户行业特性调整模块结构
- 技术选型需真实可行，不可虚构产品能力
- 方案输出后建议人工审核技术细节
- 搭配 [[2026-06-04_EROS_Agent配置#SolutionArchitect|SolutionArchitect Agent]] 使用

## 4. 标书应答Prompt

### 适用场景
在收到招标文件后，针对具体条款进行逐条应答。

### Prompt模板

```
你是一位招投标专家，擅长撰写高质量的标书应答文件。请基于以下信息，对接下来的招标要求进行应答。

## 输入信息
- 项目名称：{{project_name}}
- 招标方：{{tender_organization}}
- 招标条款：{{tender_clause}}
- 我方产品/方案：{{our_solution_description}}
- 相关资质：{{our_certifications}}
- 类似案例：{{similar_cases}}
- 特殊要求：{{special_requirements}}

## 输出要求
请按照以下结构对每个招标条款进行应答：

### 应答总体策略
- 得分点分析（哪些条款必须满分）
- 差异化点（在哪些条款可以拉开差距）
- 风险条款（需法务审核的条款）

### 逐条应答（对每个条款）

**条款编号：{{clause_id}}**
**条款原文：** {{original_text}}
**应答等级：** 完全满足 / 部分满足（需说明）/ 偏离（需说明理由）

#### 应答内容
- 我方响应方案（具体如何满足）
- 佐证材料（资质、案例、技术白皮书）
- 优势亮点（如有超出要求的部分，突出强调）

#### 潜在质疑与回应
- 评标专家可能质疑的点
- 我们的解释与说服逻辑

### 应答检查清单
- [ ] 所有条款均已响应，无遗漏
- [ ] 偏离条款有充分理由且已获内部审批
- [ ] 加分项已充分利用
- [ ] 证明材料已准备齐全
- [ ] 格式符合招标文件要求

请以"零废标"为底线，"高分中标"为目标进行应答。
```

### 使用说明
- 运行前确保已完整阅读招标文件
- 偏离条款需同步法务和销售负责人审核
- 建议先用此Prompt生成初稿，再由专家精修
- 搭配 [[2026-06-04_EROS_Agent配置#SolutionArchitect|SolutionArchitect Agent]] 和 [[2026-06-04_EROS_Agent配置#DealStrategy|DealStrategy Agent]] 使用

## 5. 拜访话术生成Prompt

### 适用场景
在客户拜访前，根据不同角色（CIO、技术总监、业务负责人等）生成定制化沟通话术。

### Prompt模板

```
你是一位顶级企业销售教练，擅长为不同客户角色定制沟通话术。请基于以下信息生成拜访话术。

## 输入信息
- 客户名称：{{customer_name}}
- 拜访对象：{{meeting_person}}（职位：{{position}}）
- 拜访目的：{{meeting_purpose}}
- 客户当前阶段：{{customer_stage}}（首次接触/需求确认/方案汇报/商务谈判）
- 我方参会人员：{{our_attendees}}
- 核心目标：{{core_objective}}
- 已知客户关切：{{known_concerns}}
- 需要回避的话题：{{topics_to_avoid}}

## 输出要求
请按照以下结构输出拜访话术：

### 1. 拜访策略摘要
- 本次拜访的核心目标（一句话）
- 客户的"买点"分析（他们为什么关心这件事）
- 预期客户的3个潜在异议及应对

### 2. 开场话术（0-3分钟）
- 破冰话题建议（基于客户行业/个人背景）
- 议程说明话术
- 价值锚点植入（一句话说明我们能带来的价值）

### 3. 需求探询话术（核心环节）
- SPIN探询框架应用
  - 现状问题（Situation）
  - 困难问题（Problem）
  - 影响问题（Implication）
  - 需求确认（Need-payoff）
- 针对{{position}}角色的专项问题清单（5-8个）
- 深度倾听提示（什么时候该追问，什么时候该沉默）

### 4. 方案呈现话术
- 核心价值主张（一句话版本 + 详细版本）
- 客户场景化故事（用客户自己的场景讲方案）
- 差异化亮点呈现（为什么选我们不是别人）

### 5. 异议处理话术库
| 常见异议 | 回应策略 | 参考话术 |
|----------|----------|----------|
| 价格太贵 | 价值重构 | ... |
| 已有供应商 | 差异化切入 | ... |
| 再考虑考虑 | 紧迫感建立 | ... |
| 技术不成熟 | 案例证明 | ... |
| 老板不批 | 向上汇报辅助 | ... |

### 6. 收尾与下一步话术
- 总结确认话术
- 下一步行动推进话术
- 如果客户犹豫的备选方案话术

### 7. 拜访后跟进
- 24小时内感谢信模板
- 会议纪要要点提示
- 下次接触的钩子设计

请确保话术自然、专业、有说服力，避免销售腔过重。
```

### 使用说明
- 拜访前48小时生成，留出演练时间
- 可根据实际拜访对象调整角色专项问题
- 建议与销售团队进行角色扮演演练
- 搭配 [[2026-06-04_EROS_Agent配置#Discovery|Discovery Agent]] 使用

## 使用说明

### Prompt使用原则
1. **变量替换**：所有 `{{variable}}` 需替换为真实业务数据
2. **上下文增强**：可在Prompt前补充行业/客户/项目的额外背景
3. **迭代优化**：首次输出后，可追加"请深化XX部分"进行二次优化
4. **人工审查**：所有AI生成内容需经过人工审核后方可用于客户交互

### 推荐工作流
```
客户信息输入 → 客户画像Prompt → 竞品分析Prompt → 技术方案Prompt → 拜访话术Prompt
                                    ↘ 标书应答Prompt（如有招标）
```
