# 云计算基础概念

## 一、概述

云计算（Cloud Computing）是一种通过互联网按需提供计算资源（服务器、存储、数据库、网络、软件等）的服务模式。用户无需自建物理基础设施，按使用量付费。根据 NIST（美国国家标准与技术研究院）定义，云计算具有五大基本特征、三大服务模式和四种部署模式。

## 二、五大基本特征（NIST 定义）

| 特征 | 说明 |
|------|------|
| **按需自助服务** | 用户可根据需要自行配置和使用计算资源，无需与服务商人工交互 |
| **广泛的网络接入** | 通过标准网络协议（HTTP/HTTPS）从各种终端设备访问 |
| **资源池化** | 计算资源以多租户模式服务众多客户，物理/虚拟资源可按需动态分配 |
| **快速弹性** | 资源可快速弹性伸缩，对客户而言可无限获取 |
| **可计量服务** | 自动控制和优化资源使用，提供透明度（按使用量计费、监控、报告） |

## 三、三大服务模式

### 3.1 IaaS — 基础设施即服务

**IaaS**（Infrastructure as a Service）：提供虚拟化的计算资源（虚拟机、存储、网络等）。

```
用户管理：      应用、数据、运行时、中间件、OS
云服务商管理：  虚拟化、服务器、存储、网络
```

典型产品：
- **AWS EC2**（弹性计算云）
- **阿里云 ECS**（弹性计算服务）
- **华为云 ECS / 腾讯云 CVM**
- **Google Compute Engine**

用户场景：
- 自主搭建开发/测试环境
- 网站托管
- 大数据分析基础设施
- 灾备与归档存储

### 3.2 PaaS — 平台即服务

**PaaS**（Platform as a Service）：提供应用开发和部署平台（运行时、中间件、数据库等）。

```
用户管理：      应用、数据
云服务商管理：  运行时、中间件、OS、虚拟化、服务器、存储、网络
```

典型产品：
- **AWS Elastic Beanstalk**
- **阿里云 EDAS / SAE**（Serverless 应用引擎）
- **Google App Engine**
- **Heroku**（早期 PaaS 先驱）
- **Cloud Foundry**（开源 PaaS）

用户场景：
- 快速应用开发和部署
- API 服务
- 微服务托管
- CI/CD 流水线

### 3.3 SaaS — 软件即服务

**SaaS**（Software as a Service）：通过互联网提供完整软件应用。

```
用户管理：      配置和少量定制
云服务商管理：  全部技术栈（应用到基础设施）
```

典型产品：
- **办公协作**：Microsoft 365、Google Workspace、钉钉、飞书
- **CRM**：Salesforce、纷享销客
- **ERP**：用友云、金蝶云
- **ITSM**：ServiceNow

用户场景：
- 企业日常办公
- 客户关系管理
- 项目管理与协作

### 3.4 责任共担模型

| 服务模式 | 用户责任 | 云服务商责任 |
|----------|----------|--------------|
| **On-Premises** | 全部 | 无 |
| **IaaS** | 应用、数据、运行时、OS | 虚拟化、服务器、存储、网络、数据中心 |
| **PaaS** | 应用、数据 | 运行时、OS、虚拟化、服务器、存储、网络 |
| **SaaS** | 配置、数据 | 全部技术栈 |

## 四、四种部署模式

### 4.1 公有云（Public Cloud）

由第三方云服务商拥有和运营，通过互联网向公众提供服务。

**代表厂商**：
- AWS（全球第一，市场份额 ~32%）
- Microsoft Azure（全球第二）
- Google Cloud Platform
- 国内：阿里云、华为云、腾讯云、百度云

**优势**：
- 零资本支出（CapEx → OpEx 转换）
- 弹性伸缩，无限扩展
- 全球化部署能力
- 运维成本低

**劣势**：
- 数据安全与合规风险
- 供应商锁定（Vendor Lock-in）
- 长期成本可能较高

### 4.2 私有云（Private Cloud）

为单一组织或企业专有使用的云环境，可托管于自有数据中心或第三方机房。

**代表方案**：
- **VMware vSphere / vCloud Suite**
- **OpenStack**（开源私有云方案）
- **华为云 Stack**（部署在客户机房的全栈方案）
- **阿里云专有云**

**优势**：
- 更高的安全性和合规性
- 可定制化程度高
- 完全控制基础设施

**劣势**：
- 高初始投入（CapEx）
- 运维复杂度高
- 扩展性受限于自有硬件

### 4.3 混合云（Hybrid Cloud）

结合公有云和私有云，允许数据和应用程序在两者间共享。

**典型场景**：
- **云爆发（Cloud Bursting）**：平时私有云承载，高峰时弹性扩展到公有云
- **数据驻留合规**：敏感数据存私有云，非敏感业务跑公有云
- **灾备架构**：私有云作为主站，公有云作为灾备

**关键技术**：
- 统一管理平台（多云管理 CMP）
- 网络互联（VPN / 专线）
- 容器平台（Kubernetes 联邦）
- 统一身份认证

### 4.4 多云（Multi-Cloud）

同时使用多个公有云服务商，避免单一供应商锁定，提升可用性。

**策略**：
- 择优选型（Best-of-Breed）
- 议价能力提升
- 容灾高可用

### 4.5 社区云（Community Cloud）

由具有共同关切的若干组织共享，支持特定社区（如政府、医疗、金融行业）。

## 五、云计算发展趋势

### 5.1 Serverless（无服务器计算）

- 无需管理服务器，按实际执行时间付费
- **FaaS**（Function as a Service）：AWS Lambda、阿里云函数计算
- **BaaS**（Backend as a Service）：Firebase、Supabase

### 5.2 边缘计算（Edge Computing）

- 将计算从中心云下沉到网络边缘
- 降低延迟、减少带宽成本
- 代表：AWS Wavelength、Cloudflare Workers、边缘云节点

### 5.3 云原生（Cloud Native）

- 以容器、微服务、服务网格、声明式 API 为核心的架构方法论
- CNCF 主导生态发展
- 代表技术栈：Kubernetes + Istio + Prometheus + Helm

### 5.4 FinOps（云财务管理）

- 将财务问责制引入云的可变支出模型
- 跨工程、财务、业务团队协作优化云成本

## 六、主流云厂商对比

| 维度 | AWS | Azure | 阿里云 | 华为云 | 腾讯云 |
|------|-----|-------|--------|--------|--------|
| **全球市占** | 32% | 23% | 中国第一 | 中国第二 | 中国第三 |
| **地域覆盖** | 全球最广 | 全球广泛 | 亚太为主 | 亚太+新兴市场 | 亚太为主 |
| **优势领域** | 全面领先 | 企业级/Office集成 | 电商/互联网 | 政企/制造业 | 游戏/社交/音视频 |
| **AI能力** | SageMaker、Bedrock | Azure AI、OpenAI 合作 | 通义大模型 | 盘古大模型 | 混元大模型 |
