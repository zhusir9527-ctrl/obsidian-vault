# 顾家家居 NeonSAN 全闪扩容实施方案

> 日期：2026-06-09 | 版本：NeonSAN-installer 2.0.17.2 / NeonSAN 3.6.1

## 背景

客户 SSD0 资源池（全闪）可用空间约 20TB。若有一台节点宕机，剩余空间不足以支撑业务资源 recovery，可能导致整体资源池使用率 100%，影响所有业务。

## 扩容范围

- **资源池**：SSD0（全闪）
- **扩容节点**：2 台全闪存储节点
- **主机名/IP**：
  - kuka1r10n00 / 192.168.250.100
  - kuka1r10n01 / 192.168.250.101

## 硬件配置

| 部件 | 规格 |
|------|------|
| CPU | Intel Xeon Silver 4314 @ 2.20GHz ×2 |
| 内存 | 32G ×2 = 64G |
| 系统盘 | SSD_480G_SATA ×2 |
| 缓存/数据盘 | NVME_4T ×8 |
| 网络 | 双口10Gb（含光模块）×2 |
| OS | QingCloud-20.04.1-AutoInstall |

## 扩容容量

| 资源池 | 资源类型 | 物理资源 | 虚拟资源 |
|--------|---------|---------|---------|
| SSD0 | 全闪节点 | 2台 × 3.84T × 8 | — |

## 关键注意事项

- 扩容与升级/维护互斥，扩容期间终止其他操作
- 操作双人复核，完成功能/性能/HA 测试后交付

## 实施步骤概要

1. 硬件上架 + BIOS 配置 + OS 安装
2. 网络配置（业务 bond + 存储双端口）
3. NeonSAN 环境检查（版本/IO Task/磁盘清理）
4. 收集硬件信息 + 节点初始化
5. 扩容 NeonSAN + 流控配置 + 一致性检查
6. 加入生产 RG + 服务验证
