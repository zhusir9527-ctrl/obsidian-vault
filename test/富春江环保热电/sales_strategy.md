# 首次拜访策略

## 拜访目标

摸清三件事：
1. 现有IT基础设施（虚拟化用什么、服务器什么品牌、存储怎么搞）
2. 信创替代的紧迫性和时间表
3. 决策链（谁说了算、预算在哪、什么时候立项）

不谈具体产品，不谈报价。以"了解需求 + 建立信任"为唯一目标。

## 关键提问

### 基础设施摸底
- "贵司现在服务器虚拟化用的是哪家？VMware 还是其他的？"
- "VMware 被博通收购后 license 续费有没有涨价？"
- "服务器大概多少台？主要跑哪些业务系统？"
- "存储用的什么方案？SAN 还是本地盘？"

### 信创摸底
- "信创替代方面，国资委有没有给时间表？"
- "操作系统和数据库有没有国产化计划？"
- "上次等保测评是什么时候？有没有整改要求？"

### 多厂区摸底
- "总部和下面电厂IT是统一管理还是各自独立？"
- "电厂那边有没有IT人员？出了问题怎么处理？"

### 痛点挖掘
- "日常运维最大的困扰是什么？"
- "系统出问题一般怎么处理？有固定的服务商吗？"
- "最近有没有什么IT方面的规划或者头疼的事？"

## 推荐切入场景

### 场景1：VMware替代 + 超融合升级（最高优先级）

话术：
"像富春江环保这样的中型国企，最近都在面临一个共同问题——VMware被博通收购后license涨了好几倍。青云的云易捷超融合方案，3个节点就能起步，一套平台同时管理虚拟机和容器，还能满足信创要求。最关键的是，TCO比VMware续费三年还便宜。"

产品：云易捷超融合（计算+存储+网络一体化）

### 场景2：信创全栈替换（合规刚需）

话术：
"2027年信创全面完成是硬指标。青云的信创虚拟化已经适配了飞腾、鲲鹏、海光、兆芯这些国产CPU，还有麒麟、统信操作系统。最关键是迁移过程我们全程驻场，保证业务不中断。"

产品：信创云易捷 + 信创虚拟化

### 场景3：多厂区统一云管（如果多厂区）

话术：
"如果总部富阳和下面几个电厂IT各自管各自的，运维压力肯定大。青云做一套轻量化的统一管理平台，总部就能看到所有电厂的IT运行状态，出了问题自动告警。"

产品：企业云 + 云易捷

## 常见异议应对

| 异议 | 应对 |
|------|------|
| "我们IT很简单，没什么需求" | "越是简单越怕出问题。VMware一旦宕机，蒸汽管网的调度系统受影响，损失可比IT预算大多了。" |
| "预算不够" | "云易捷3节点起步，比VMware续费便宜。关键是信创合规——现在不换，到2027年临时抱佛脚，成本更高。" |
| "我们有固定合作方" | "不用替换他们，我们可以和您的服务商合作。他们做集成，我们提供产品和原厂技术支持。" |
| "老板不重视IT" | "蒸汽管网是你们的生命线，IT是管网的神经系统。我们可以做个简单的TCO对比，让老板看清楚三年能省多少钱。" |

## 注意事项

- 不要穿西装打领带，环保企业偏务实，商务休闲即可
- 不要主动谈钱，等对方问到再说"比VMware便宜"
- DCS/SCADA生产系统绝对不要碰（容易踩雷）
- 富阳离杭州市区约40公里，提前出发
- 国企人员一般上午9:00-11:00状态最好
- 实控人是南昌国资委，重大决策可能不在杭州——不要追问过深
