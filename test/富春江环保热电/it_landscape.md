# IT 现状（推测）

## 基础设施

| 类别 | 推测现状 | 可能痛点 |
|------|----------|----------|
| 服务器 | 物理服务器为主，少量VMware虚拟化 | VMware license涨价，运维复杂 |
| 存储 | SAN/NAS传统存储 | 扩容困难，性能瓶颈 |
| 网络 | 传统三层架构 | 东西向流量不可视 |
| 操作系统 | Windows Server + 少量Linux | 信创要求替换为麒麟/统信 |
| 数据库 | SQL Server / Oracle | 信创迁移压力 |

## 应用系统

| 系统 | 推测 | 信创替换难度 |
|------|------|-------------|
| ERP | 用友NC / 金蝶K3 | 中（有国产替代方案） |
| DCS/SCADA | 西门子/ABB等工控系统 | 高（生产系统不能碰） |
| OA | 泛微/蓝凌 | 低 |
| 邮件 | Exchange / 腾讯企业邮 | 低 |

## 关键痛点预判

1. VMware博通收购后license成本飙升（国企普遍反映）
2. 信创2027大限倒逼替换Windows+SQL Server
3. DCS生产数据与ERP经营数据未打通（数据孤岛）
4. 多厂区（总部富阳+各地电厂）IT各自为政
5. IT团队小，运维压力大，出问题只能找集成商
6. 光伏/固废站点需要边缘计算但缺乏统一方案
