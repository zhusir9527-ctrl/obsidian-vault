# 振石集团 AI基础设施建设项目

## 项目概况

- **项目名称**: 振石控股集团AI算力基础设施建设项目
- **招标内容**: GPU算力服务器(1台H200级别8卡服务器) + 智能调度服务平台
- **关键要求**: 数据全程本地化、无厂商绑定、30天工期、三大场景性能验证
- **竞品**: 联想/DELL/华三/浪潮/超聚变(硬件)、智谱/腾讯云(平台)

## 关键技术要点

### 硬件
- 1台 NVIDIA HGX-Hopper-8GPU模组服务器(H200级别，单卡≥141GB显存，≥4.8TB/s带宽)
- Intel第五代处理器，24×64GB DDR5，7.68TB NVMe ×2
- 25GE网卡，N+1冗余电源风扇

### 软件平台要求
- 异构算力统一纳管、Kubernetes容器化调度
- GPU虚拟化/MIG切分、显存隔离
- 模型全生命周期管理、推理引擎(vLLM/SGLang/TensorRT-LLM等)
- API计量计费(秒级)、多租户配额管理
- GPU/LLM监控可观测、RBAC权限控制
- OpenAI兼容API

### 性能验证(三大场景)
- 部署模型: Qwen3.5-397B-A17B-FP8
- 场景1: 知识问答(输入8192/输出1024/并发64)
- 场景2: AI编程(输入65536/输出1024/并发32)  
- 场景3: 文本创作(输入1024/输出4096/并发128)

### 授权要求
- 无品牌锁定、无设备排他性捆绑
- 支持第三方中标品牌算力设备无缝集成

### 进度要求
- 整体工期30天
- 上线后60-90天平稳运行后验收
- 超出工期按总金额1%/天扣罚

## 竞品分析
- 智谱/腾讯云仅为Token账户充值(各5000元)，非平台产品
- 硬件推荐品牌为硬件制造商，青云以软件平台+集成服务切入
- 青云优势：软硬解耦、不绑定硬件、Kubernetes原生架构、全栈AI平台能力

## 技术偏离响应与竞争分析

- [振石集团-技术标评分表-偏离响应与竞争分析](file:///Users/zhu/Desktop/振石/振石集团-技术标评分表-偏离响应与竞争分析.docx)

### 竞品AIStation核心弱点
1. **硬件绑定**: AIStation与浪潮服务器深度绑定，本项目推荐品牌为联想/DELL/华三/超聚变，兼容性存疑
2. **国产芯片适配不足**: 对昇腾910B/海光DCU适配深度不及青云(vLLM-Ascend/MindIE/DTK)
3. **纯软件交付能力**: AIStation多与浪潮硬件捆绑销售，缺乏独立软件交付经验
4. **运营基因缺失**: 缺乏公有云级别的计量计费/多租户运营能力
5. **架构开放性**: 自研调度层与K8s原生生态兼容性受限

### 青云六大竞争壁垒
1. 无厂商绑定 - 运行在任意品牌服务器
2. 纯K8s原生 - 100%兼容CNCF生态
3. 国产芯片全面适配 - 昇腾/海光/NVIDIA三架构
4. 公有云运营基因 - 成熟的计量计费/多租户体系
5. 科创板上市 - 财务透明，原厂直服
6. 存储生态开放 - 支持10+第三方存储系统

- **官网**: aistation.ieisystem.com
- **目标行业**: 金融/教科研/交通/企业/IDC（**未强调制造业**）

### 关键数据（官方声明）
- 算力利用率提升20%
- 千卡规模GPU加速比90%
- 推理速度最大提升300%
- 有效训练时间占比90%
- 集成10+开源大模型

### 针对振石项目的6大不适用点
1. 硬件绑定 → 推荐品牌为联想/DELL/华三/超聚变，AIStation兼容性存疑
2. 国产芯片适配不足 → 官网未展示昇腾/海光DCU适配
3. 制造业案例缺失 → 官网解决方案未覆盖制造业
4. 运营计费能力薄弱 → 官网未展示计费/账单/多租户配额模块
5. Agent非招标重点 → V5.4核心亮点(OpenClaw)与招标需求不匹配
6. 千卡优化无意义 → 本项目仅1台8卡服务器

### 竞争策略
- 青云7:0优势维度（架构/兼容/国产/运营/制造/交付/多租户）
- 承认AIStation真优势（训练稳定性/推理加速/大模型微调）但弱化其在项目中的相关性
- 主动提出性能实测对比，用数据说话
